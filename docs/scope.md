# Scope

## Project Name
BuiltExpert Marketing Website

## Scope Summary
This document defines what is included, what is excluded, the expected implementation sequence, and the operating constraints for the BuiltExpert marketing website project.

The goal is to keep the project focused on launching a credible, conversion-oriented marketing site without letting it sprawl into unrelated software or premature complexity.

## In Scope

### Core Website Pages
- Home
- Services
- Audit
- Work / Case Studies
- Process
- Pricing
- About
- Contact
- Thank You / Confirmation
- Privacy Policy

### Recommended Supporting Content
- service content structure
- audit offer content structure
- case study content structure
- FAQ content structure
- contact and qualification form structure

### Functional Deliverables
- responsive marketing website
- clear CTA pathways
- lead form implementation
- booking CTA integration
- editable content setup
- analytics setup for core conversion events
- SEO baseline implementation
- staging / preview workflow
- QA before launch

### Technical Deliverables
- Next.js app setup
- React and TypeScript implementation
- Tailwind CSS integration
- reusable section/component structure
- maintainable content architecture
- production build and deployment readiness

## Out Of Scope

### Not Included In This Phase
- custom CRM development
- advanced contractor portal or customer app functionality
- member accounts or gated dashboards
- full blog production pipeline beyond future-readiness
- advanced automation systems
- complex third-party enterprise integrations
- multilingual implementation
- full-scale CRO experimentation program
- server-side tracking and advanced attribution modeling
- broad niche expansion beyond planned structure
- unrelated SaaS product development

### Also Out Of Scope
- brand identity redesign
- visual design system definition
- design-direction decisions already handled elsewhere
- unrelated internal tools

## Deliverables
At the end of the scoped project, the team should have:

- a launched marketing website or production-ready build
- core pages complete
- forms and key CTA paths working
- analytics events configured at baseline
- core SEO foundations implemented
- documentation sufficient for handoff and future iteration

## Proposed Timeline

### Phase 1 — Strategy And Content Alignment
- finalize sitemap
- align positioning and messaging
- confirm core offers
- confirm CTA strategy
- gather or finalize content inputs

### Phase 2 — Structure And Build
- scaffold project
- implement reusable sections/components
- build homepage
- build core inside pages
- connect forms and CTA destinations

### Phase 3 — SEO, Analytics, And QA
- implement page metadata
- validate internal linking
- configure event tracking
- test forms, links, and core flows
- run responsive and cross-browser QA

### Phase 4 — Launch
- final review
- production deployment
- tracking validation
- post-launch checks

## Constraints

### Business Constraints
- the site must support lead generation first
- messaging must be clear enough for non-marketing-savvy contractors
- the site should filter out low-fit leads without sounding hostile
- the build should support future niche expansion without requiring a full rebuild

### Product Constraints
- no design decision conflicts with separate design documentation
- no unnecessary feature sprawl
- no overengineering for phase-one needs
- content structure should remain maintainable by one operator or a small team

### Technical Constraints
- stack should remain simple and pragmatic
- dependencies should be limited to what is actually needed
- implementation should be maintainable and easy to extend
- performance and reliability must not be sacrificed for unnecessary complexity

## Assumptions
- BuiltExpert positioning is already directionally defined
- design decisions are documented separately if needed
- the main conversion goal is discovery calls and lead submissions
- content inputs will be available or can be created during the project
- the website will be used as a foundation for future service and campaign expansion

## Risks
- scope creep into SaaS or internal tool ideas
- weak proof assets reducing trust
- inconsistent content inputs
- unclear niche hierarchy causing visitor confusion
- delays caused by trying to perfect non-essential details before launch

## Scope Control Principle
When decisions come up, the default filter should be:

**Does this help the website launch as a clearer, higher-converting marketing site for contractors right now?**

If not, it should likely move to a later phase.
