# Content Ops

## Purpose
This document defines how content is updated, reviewed, published, and maintained for the BuiltExpert marketing website after launch.

---

## 1. Content Ops Goals
The workflow should:
- keep core pages accurate
- make updates repeatable
- reduce publishing errors
- protect SEO structure
- keep proof and offers fresh
- support future growth without chaos

---

## 2. Content Ownership

| Content Area | Primary Owner | Secondary Owner | Notes |
|---|---|---|---|
| Core marketing pages | Owner | Editor/Builder | |
| Service pages | Owner | Editor | |
| Audit page | Owner | Editor | |
| Case studies | Owner | Editor | |
| Pricing content | Owner | — | High-sensitivity |
| SEO metadata | Owner | SEO support if used | |

---

## 3. Publishing Workflow
1. Draft content
2. Internal review
3. Final approve
4. Publish
5. QA
6. Verify metadata and internal links

---

## 4. Update Types

### Minor Update
Examples:
- typo fixes
- CTA wording tweak
- testimonial swap
- metadata adjustment

### Major Update
Examples:
- service repositioning
- pricing changes
- homepage messaging rewrite
- new case study launch
- new trade page

---

## 5. Case Study Workflow
1. Identify project as case-study candidate
2. Gather summary, outcomes, proof assets
3. Draft short project summary
4. Draft full case study if warranted
5. Review for sensitivity/approvals if needed
6. Publish and internally link relevant pages

---

## 6. SEO Update Workflow
When required:
- new page published
- page title changes
- slug changes
- internal linking changes
- service repositioning
- case study added

Checklist:
- title tag updated
- meta description updated
- heading structure reviewed
- internal links added
- schema considered if needed
- redirects added if URL changed

---

## Recommendation
Keep content ops lightweight but disciplined. The main risk is stale proof, inconsistent updates, and undocumented changes.
