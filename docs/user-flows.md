# User Flows

## Purpose
This document outlines the key user journeys through the BuiltExpert marketing website. These flows define how visitors should move from awareness to action, and where the site needs to reduce friction, answer objections, and create confidence.

These are marketing-site flows focused on lead generation, trust-building, and qualification.

---

## Flow 1: Homepage To Discovery Call

### User Goal
The contractor wants to quickly understand what BuiltExpert does and decide whether to book a call.

### Entry Points
- direct traffic
- referrals
- branded search
- outbound links
- social profile links

### Steps
1. User lands on the homepage
2. User scans hero section to understand offer and positioning
3. User reviews proof elements and service overview
4. User checks supporting trust sections like work, process, or FAQ
5. User clicks primary CTA
6. User is taken to booking link, contact page, or qualified form
7. User completes action
8. User reaches thank-you or confirmation state

### Key Friction Points
- unclear value proposition
- not enough contractor relevance
- weak proof
- too much agency-style fluff
- unclear next step

### Success Condition
- booking completed
- or qualified inquiry submitted

---

## Flow 2: Homepage To Audit Offer

### User Goal
The user wants a lower-risk entry point before committing to a larger project.

### Entry Points
- homepage
- referrals
- outbound traffic
- internal CTA blocks

### Steps
1. User lands on homepage
2. User sees audit-related CTA or navigation path
3. User visits Audit page
4. User reviews what the audit includes and why it matters
5. User decides whether to inquire or book
6. User submits or books

### Success Condition
- contractor chooses the audit path instead of bouncing

---

## Flow 3: Services To Contact

### User Goal
The user needs more detail before contacting BuiltExpert.

### Steps
1. User lands on Services page
2. User scans top-level services overview
3. User reviews one or more service categories
4. User determines relevance and fit
5. User clicks CTA to contact or book a call
6. User submits form or books

### Questions The User Is Asking
- do they offer what I need?
- do they understand contractor marketing?
- is this too big or too small for me?
- what happens after I reach out?

### Success Condition
- user transitions from browsing to direct inquiry

---

## Flow 4: Service Detail Page To Contact

### User Goal
The user wants to validate one specific service before taking action.

### Entry Points
- organic search
- paid campaigns
- internal links
- shared direct links

### Steps
1. User lands on a service page
2. User reads the service promise and who it is for
3. User reviews outcomes, process, and scope framing
4. User checks proof or related work if available
5. User sees CTA tied to that service
6. User clicks through to contact or booking
7. User completes inquiry

### Success Condition
- qualified contact intent generated from a service-specific page

---

## Flow 5: Work / Case Study To Contact

### User Goal
The user wants proof before reaching out.

### Steps
1. User lands on Work overview or a case study page
2. User scans project relevance
3. User reads project summary, approach, and results
4. User builds trust through proof
5. User clicks CTA to contact or book
6. User completes inquiry

### Success Condition
- trust is strong enough to trigger conversion

---

## Flow 6: Pricing To Contact

### User Goal
The user wants to understand investment range and fit before reaching out.

### Steps
1. User lands on Pricing page
2. User reviews engagement types or ranges
3. User understands how scope changes pricing
4. User self-qualifies
5. User clicks CTA to discuss project
6. User submits inquiry or books call

### Success Condition
- better-fit leads move forward
- poor-fit leads self-select out

---

## Flow 7: Contact Form Submission

### User Goal
The user is ready to reach out and wants a clear low-friction way to do it.

### Steps
1. User lands on Contact page or form section
2. User scans short intro and what happens next
3. User fills out qualification form
4. User submits
5. User sees success state or thank-you page
6. User understands next step in the process

### Required Form Experience
- short enough to complete quickly
- structured enough to qualify the lead
- confidence-building language around response expectations
- clear submission confirmation

### Success Condition
- form submitted successfully
- user knows what happens next

---

## Global Flow Principles

1. Every core page needs a next step  
2. Trust and contractor relevance must show up early  
3. The site should qualify, not just attract  
4. Contact must feel easy  
5. Confirmation matters after every conversion action
